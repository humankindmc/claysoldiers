# Security Policy

Please do not open public issues for private security vulnerabilities.

If GitHub private vulnerability reporting or security advisories are enabled for this repository, use that path to contact the maintainers privately.

If private advisories are not available, contact the maintainer listed in the repository through their GitHub profile or other maintainer contact listed by the project.

When reporting a vulnerability, include:

- Affected ClaySoldiers version or commit
- Paper and Java versions
- Steps to reproduce
- Impact and expected severity
- Any logs, stack traces, or proof-of-concept details that help maintainers validate the issue

Maintainers will review security reports privately before public disclosure.
